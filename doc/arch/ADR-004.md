# ADR-004: Chamber pertenece a Diligencia

| Campo | Valor |
|---|---|
| **Decisión** | Chamber es dominio de Diligencia. Todo código, UI, tray, skills y configuración en openchamber pertenecen a Diligencia y se rigen por sus reglas. |
| **Estado** | Accepted |
| **Fecha** | 2026-06-27 |
| **Supersedes** | N/A |
| **Superseded by** | N/A |
| **Impacto** | AGENTS.md §Identidad, MANDATO.md M6, $CHAMBER variable |

## Contexto

Chamber era tratado como un proyecto externo bajo "tutela" de Diligencia. Esto creaba ambigüedad en el alcance de las reglas: ¿las modificaciones en Chamber siguen PLAN→BUILD? ¿/commit o /CBP en Chamber? La respuesta era confusa. Se necesitaba una regla clara.

## Decisión

Diligencia ahora declara: "Chamber es mi interfaz. No es un proyecto separado — todo lo que pasa en Chamber me pertenece." Chamber sigue las mismas reglas que Diligencia:
- PLAN→BUILD para cambios
- /commit, /CBP, /version para git
- Mismo flujo de circuitos y skills

## Consecuencias

- Chamber ya no es "proyecto adaptado" — es parte del cuerpo de Diligencia
- Cualquier modificación en Chamber debe documentarse en CHANGELOG de Diligencia
- $CHAMBER variable registrada en AGENTS.md
- MANDATO.md M6 establece: "Chamber pertenece a Diligencia"
- Contra: se pierde la separación conceptual Diligencia↔Chamber
