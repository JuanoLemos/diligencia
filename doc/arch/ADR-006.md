# ADR-006: /doctor renombrado a /salud

| Campo | Valor |
|---|---|
| **Decisión** | Renombrar el comando /doctor de Diligencia a /salud para evitar conflicto con el comando nativo /doctor de OpenCode. |
| **Estado** | Accepted |
| **Fecha** | 2026-06-28 |
| **Supersedes** | N/A |
| **Superseded by** | N/A |
| **Impacto** | salud.md, CBP.md, AGENTS.md, COMANDOS.md, versión global |

## Contexto

OpenCode tiene un comando nativo `/doctor` (built-in, sin archivo .md). Diligencia también define `/doctor`. OpenCode resuelve prioridades como: local > global > nativo. El global de Diligencia siempre ganaba, haciendo imposible acceder al `/doctor` nativo de OpenCode. Además, el nombre "doctor" era confuso: el comando no diagnostica — hace cuidado integral del proyecto.

## Decisión

Renombrar `/doctor` de Diligencia a `/salud`. El archivo `doctor.md` pasa a `salud.md` en los 3 niveles (local, global, Chamber). Todas las referencias internas (CBP.md, AGENTS.md, COMANDOS.md) se actualizan. El espacio de nombre `/doctor` queda disponible para el nativo de OpenCode si se necesita.

## Consecuencias

- Se resuelve el conflicto con OpenCode nativo
- El nombre `/salud` es más descriptivo del comando real (cuidado integral del proyecto, no diagnóstico)
- 31 referencias en CBP.md actualizadas
- Recursos: `doctor.md` deprecado, `salud.md` activo en 3 repos (Diligencia, global, Chamber)
