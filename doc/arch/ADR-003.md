# ADR-003: Estructura estándar de archivos

- **Fecha:** 2026-05-08
- **Estado:** Aceptado

## Contexto

Cada proyecto organizaba su documentación de manera distinta. Némesis usaba `autoridad/`, MarketAI usaba `docs/`, otros proyectos podían usar `documentacion/`. Esto impedía:
- Navegar intuitivamente entre proyectos
- Compartir comandos entre proyectos
- Automatizar backups y verificaciones

## Decisión

Definir una estructura única que todos los proyectos adaptados siguen:

```
<raiz>/
├── ROADMAP.md                    ← Estrategia y plan
├── CHECKLIST.md                  ← Tareas pendientes
├── CHANGELOG.md                  ← Historial de versiones
├── AGENTS.md                     ← Mapeo de variables + comandos
├── DILIGENCIA.md                 ← Sello de metodología
├── .markdownlint.json            ← Reglas de linting
├── doc/
│   ├── arch/                     ← ADRs, sistema, bitácora
│   ├── guias/                    ← Guías de usuario (plural)
│   └── mecanicas/                ← Mecánicas del juego (si aplica)
└── .opencode/
    └── commands/                 ← Comandos específicos del proyecto
```

## Reglas

1. **Solo 3 archivos en raíz:** ROADMAP.md, CHECKLIST.md, CHANGELOG.md. AGENTS.md y DILIGENCIA.md también viven en raíz pero son de sistema.
2. **doc/arch/:** Archivos de arquitectura (ADRs), sistema, bitácora.
3. **doc/guias/:** Plural forzado. Si existía `doc/guia/`, se renombra.
4. **doc/mecanicas/:** Solo si el proyecto define mecánicas (juegos, narrativa).

## Nombres de archivo

- `ROADMAP.md` → siempre mayúsculas
- `CHECKLIST.md` y `CHANGELOG.md` → mayúsculas, sin espacios
- ADRs: `ADR-NNN.md` con tres dígitos y guión
- Guías: prefijo `GUIA_DE_` (ej. `GUIA_DE_USO.md`)

## Consecuencias

- Positivas:
  - Navegación predecible entre proyectos
  - Comandos portables (misma estructura = mismas variables)
  - Backups simples: copiar raíz + doc/ + .opencode/

- Negativas:
  - Migrar estructura existente requiere mover archivos
  - Rompe enlaces/documentos externos que apuntaban a la vieja estructura

## Proyectos que siguen esta estructura

- Némesis Detective (migrado v1.0)
- MarketAI (migrado v1.0)
- Diligencia (proyecto documental)
