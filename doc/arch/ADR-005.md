| Campo | Valor |

## Contexto

La cantidad de comandos Diligencia había crecido a 39. Muchos eran redundantes: /bug y /incidente hacían lo mismo, /checklist duplicaba funcionalidad de /rm y /next, /salud era un BUILD* sin PLAN que heredaba datos de /doctor. Se identificaron solapamientos en análisis del consejero.

## Decisión

Fusionar (no eliminar) los comandos redundantes absorbiendo su funcionalidad:

| Fusión | Resultado |
|---|---|
| /bug + /incidente | → /reportar --tipo bug|incidente |
| /checklist | → /rm (inconsistencias) + /next (priorización) |
| /salud | → /doctor Fase 3h (status-salud.md) |
| /pushgh | → /commit --push |
| /report | → /estado --full |
| /backupall | → /backup --all |
| /legal | → /doctor Fase 1h |
| /+guia + /upguia + /+mec + /upmec | → /doc --tipo guia|mecanica |

## Consecuencias

- 39→28 comandos (11 fusionados, 0 perdidos)
- CBP.md actualizado: /salud y /pushgh eliminados del circuito
- 18 comandos deprecados en .old/commands/
- Menor carga cognitiva para nuevos usuarios
