# ADR-001: Arquitectura de dos capas de comandos

- **Fecha:** 2026-05-08
- **Estado:** Aceptado
- **Supersedes:** N/A
- **Superseded by:** N/A

## Contexto

Cada nuevo proyecto OpenCode requería copiar comandos como `/commit`, `/plan`, `/health`. Esto generaba:
- Duplicación de código
- Inconsistencia entre versiones del mismo comando
- Mantenimiento costoso

## Decisión

Implementar una arquitectura de dos capas:

1. **Capa global** (`~/.config/opencode/commands/`) — comandos universales heredados automáticamente por todos los proyectos
2. **Capa local** (`<proyecto>/.opencode/commands/`) — comandos específicos del proyecto, con prioridad sobre globales

Los comandos globales **no se copian** al proyecto. OpenCode los resuelve automáticamente: busca primero en local, luego en global.

## Consecuencias

- Positivas:
  - Un solo lugar para mantener comandos universales
  - Proyecto nuevo arranca con `/commit`, `/plan`, `/health` sin configuración
  - Cada proyecto solo escribe comandos que lo diferencian

- Negativas:
  - Dependencia de la carpeta global (si se pierde, se pierden comandos)
  - Confusión potencial sobre qué comandos están disponibles

## Alternativas consideradas

- **Copia al crear proyecto**: Descartada por duplicación y mantenimiento.
- **Un solo nivel (global)**: Descartado porque proyectos necesitan comandos específicos.
